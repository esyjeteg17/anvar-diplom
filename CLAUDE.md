# CLAUDE.md — Anvar (проект SBORIKA)

Документация и правила для агентов и разработчиков, работающих в этом репозитории.

## О проекте

Дипломный проект студента 4-го курса — сайт для школы робототехники **SBORIKA**.
Состоит из двух частей: публичный лендинг + админ-панель.

### Бренд

- **Название**: SBORIKA — школа инженерного мышления для детей.
- **Слоган**: «Собирай. Думай. Создавай.»
- **Возраст детей**: 5–14 лет.
- **Формат занятий**: групповые, офлайн, в партнёрских центрах Москвы.
- **Контакты**: +7 (919) 142-61-66, ВКонтакте (есть QR-код).
- **Визуальный стиль**: шестерёнки как главный декоративный элемент.

На занятиях дети:
- собирают модели из конструктора;
- программируют роботов с планшета;
- развивают логику и инженерное мышление.

Формат:
- набор и планшет на каждого ребёнка;
- игровой и понятный формат подачи;
- не требует никакой подготовки.

### Лендинг — структура страницы (по порядку)

Одна страница со следующими секциями:

1. **Navbar** — логотип SBORIKA (текст + шестерёнка вместо буквы O), навигация по якорям, кнопка «Записаться».
2. **Hero** — главный экран: заголовок «Робототехника для детей», подзаголовок «5–14 лет», CTA-кнопка «Пройти квиз и получить бесплатное занятие», декоративные шестерёнки.
3. **О школе** — миссия, ценности, коротко о SBORIKA.
4. **Занятия** — что делают дети, формат, преимущества.
5. **Центры** — «Мы работаем в партнёрских центрах Москвы» (карточки с заглушками, потом заменяются реальными адресами).
6. **Квиз** — 10 вопросов с вариантами ответов, логика выдачи промокода.
7. **Контакты** — телефон, ВКонтакте, форма обратной связи.

### Админ-панель

Назначение — управлять контентом лендинга и заявками (точная схема согласовывается по ходу).

### Дизайн / палитра

Цвета вынесены в Tailwind 4 `@theme` в `frontend-landing/app/assets/css/main.css`. Использовать **только** через брендовые классы (`bg-brand`, `text-ink`, `border-line` и т.д.), не хардкодить hex в компонентах.

| Токен | Hex | Назначение |
|---|---|---|
| `bg` | `#F5F5F5` | основной фон страницы |
| `brand` | `#EA5C1F` | акценты, CTA, оранжевые шестерёнки |
| `line` | `#D6D7D7` | разделители, рамки, нейтральный серый |
| `slate` | `#4F5D75` | сине-серые панели, иконки, secondary |
| `ink` | `#212431` | основной тёмный текст, тёмные блоки |

Графика:
- Главный декоративный мотив — **шестерёнки** (оранжевые / сине-серые / белые) разных размеров. Используем как фоновые декоративные элементы, в логотипе (буква «O» в SBORIKA), в иконках секций.
- Поддерживающий мотив — **изометрические LEGO-кирпичики**.
- Hero-блоки могут содержать **наклонные сине-серые срезы/ленты** (как на постере).
- Логотип SBORIKA — жирный гротеск, буква «O» заменена шестерёнкой.

## Стек (обязательный)

Любые изменения в проекте **должны** оставаться в рамках этого стека. Не вводить альтернативные фреймворки/библиотеки без явного согласия пользователя.

### Frontend

- **Nuxt 4** (Vue 3, `app/` structure).
- **TypeScript** в strict-режиме.
- **Tailwind CSS v4** через `@tailwindcss/vite` плагин (Tailwind 4 не использует `tailwind.config.js` по умолчанию — конфигурация через CSS `@theme`).
- **Pinia** через модуль `@pinia/nuxt`.
- Запросы к API — встроенный `$fetch` / `useFetch` Nuxt-а; базовый URL берётся из `runtimeConfig.public.apiBase`.

### Backend

- **Python 3.12+**, **Django 5.x**.
- **Django REST Framework** для API.
- **PostgreSQL** через `psycopg` (v3).
- `django-cors-headers` для CORS.
- `django-environ` для конфигурации через `.env`.
- Кастомные приложения живут в каталоге `apps/` и регистрируются как `apps.<name>` в `INSTALLED_APPS`.

### База данных

- PostgreSQL, локально.
- Имя БД: `anvar_db`
- Пользователь: `postgres`
- Пароль: `postgres`
- Хост/порт по умолчанию: `127.0.0.1:5432`
- Эти значения переопределяются через `backend/.env`.

## Запуск (dev)

### 1. PostgreSQL

Убедиться, что локальный PostgreSQL запущен и есть база `anvar_db`:

```bash
createdb -U postgres anvar_db
# или в psql: CREATE DATABASE anvar_db OWNER postgres;
```

### 2. Backend

```bash
cd backend
source venv/bin/activate
python manage.py migrate
python manage.py createsuperuser   # опционально
python manage.py runserver 0.0.0.0:8042
```

API: `http://localhost:8042/api/`
Health-check: `http://localhost:8042/api/health/`
Django admin: `http://localhost:8042/admin/`

### 3. Frontend — лендинг

```bash
cd frontend-landing
npm install
npm run dev    # http://localhost:4040
```

### 4. Frontend — админ-панель

```bash
cd frontend-admin
npm install
npm run dev    # http://localhost:4041
```

## Конвенции

### Backend

- Новые сущности — отдельные приложения в `backend/apps/<name>/`. Регистрируются в `INSTALLED_APPS` как `apps.<name>`.
- API-роуты подключаются в `config/api_urls.py` через `path("<resource>/", include("apps.<name>.urls"))`.
- Все API-эндпоинты — под префиксом `/api/`.
- Секреты, ключи, креды БД — только через `.env` и `django-environ`. Не хардкодить.
- Часовой пояс по умолчанию — `Europe/Moscow`, локаль — `ru-ru`.

### Frontend

- Структура — Nuxt 4 (`app/` верхнеуровневый каталог).
- Стили — только Tailwind utility-классы. Если нужен общий стиль — использовать `@apply` в `assets/css/main.css` или Tailwind `@theme` для токенов.
- State management — только Pinia, store-ы в `app/stores/`.
- Запросы к API — `useFetch` / `$fetch` с базой `useRuntimeConfig().public.apiBase`.
- Не добавлять Nuxt UI / другие UI-киты без согласования.
- Никаких эмодзи в продуктовом UI без явной просьбы.

#### Компонентный подход (обязательно)

- **Максимальная декомпозиция**: каждый самостоятельный элемент UI — отдельный компонент. Кнопки, карточки, инпуты, секции, заголовки — всё выносится в `app/components/`.
- **Переиспользование через пропсы**: если компонент чуть-чуть отличается в разных местах (цвет, размер, текст, вариант) — не дублировать, а параметризовать через `props`. Новый компонент — только если логика принципиально другая.
- **Структура компонентов**: группировать по смыслу (`components/ui/`, `components/layout/`, `components/sections/` и т.д.), не по страницам.
- Любой повторяющийся UI-паттерн (≥2 раза) — сигнал к выделению компонента.

### Общие

- **Все комментарии в коде — на русском языке** (требование диплома, важно).
- Названия БД, env-переменных, портов — единообразны: backend на 8042, landing на 4040, admin на 4041.
- Не коммитить `.env`, `node_modules/`, `venv/`, `__pycache__/`.
- Для production-настроек делать отдельный профиль; не править `.env.example` под локальные нужды.
